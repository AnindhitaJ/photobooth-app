LUX PHOTOBOOTH — CAMERA + STORAGE FIX
=====================================

File yang berubah:
- camera.html
- filter.html
- result.html
- api/cleanup.js
- sw.js

Perubahan utama:
1. Capture mempertahankan resolusi kamera sampai maksimum 1920x1440 dan tidak melakukan upscaling.
2. Foto besar disimpan satu kali di IndexedDB; localStorage hanya fallback.
3. Live photo dipindahkan dari localStorage ke IndexedDB.
4. Foto mentah dihapus setelah filteredPhotos berhasil disimpan dan diverifikasi.
5. Seluruh temporary IndexedDB dibersihkan ketika memulai sesi kamera baru atau memilih Foto Lagi.
6. Filter normal tidak mengompresi ulang foto.
7. Cleanup Supabase membaca created_at, mendukung nama file userId_timestamp_filename, dan memakai pagination.
8. Versi cache service worker dinaikkan agar perangkat tidak terus memakai HTML lama.

Deploy:
- Upload seluruh isi folder ini ke project/deployment yang sama.
- Untuk cleanup Supabase yang aman, tambahkan environment variable Vercel:
  SUPABASE_SERVICE_ROLE_KEY=<service-role key dari Supabase>
  CRON_SECRET=<secret acak>
- Jangan pernah menaruh service-role key di file HTML atau JavaScript frontend.

Catatan:
- Tanpa SUPABASE_SERVICE_ROLE_KEY, cleanup masih mencoba key fallback lama. Bila policy Storage tidak mengizinkan delete dengan anon key, cron akan gagal; fitur kamera tetap berjalan normal.
- Setelah deploy, buka aplikasi sekali dengan koneksi internet agar service worker v1.9 terpasang. Bila perangkat tetap menampilkan versi lama, tutup semua tab aplikasi lalu buka kembali.
