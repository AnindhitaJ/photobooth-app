LUX PHOTOBOOTH — IPAD MULAI EVENT FIX V3
========================================

GEJALA
- Tombol “Mulai” di halaman Event terlihat bisa ditekan, tetapi iPad tidak berpindah halaman.
- Biasanya terjadi pada event yang memakai welcome screen atau setelah beberapa sesi foto.

AKAR MASALAH
- event.html menyimpan seluruh objek event ke localStorage.
- welcome_screen_url dapat berupa data URL PNG berukuran beberapa MB ketika upload Storage fallback.
- Nilai welcome itu disimpan lagi pada key terpisah, sehingga data besar diduplikasi.
- Safari/iPad melempar QuotaExceededError secara sinkron. Karena error tidak ditangkap, baris navigasi tidak pernah dijalankan.

FILE YANG DITIMPA
1. event.html
2. event-run.html
3. template.html
4. result.html
5. sw.js

CARA PASANG
1. Timpa kelima file di root repository.
2. Commit dan deploy ulang ke Vercel.
3. Hapus PWA lama dari Home Screen iPad bila terpasang.
4. Hapus Website Data untuk domain aplikasi di Settings > Safari > Advanced > Website Data.
5. Buka ulang aplikasi dan buat/mulai event kembali.

PERUBAHAN
- Welcome screen besar tidak lagi disalin ke localStorage.
- Metadata event yang disimpan hanya ID, nama, template IDs, dan status overlay.
- sessionStorage dipakai sebagai jalur utama dan localStorage sebagai kompatibilitas/fallback.
- event-run mengambil welcome screen langsung dari Supabase berdasarkan event ID URL.
- Semua operasi Web Storage dibungkus try/catch sehingga kegagalan storage tidak membatalkan navigasi.
- Payload foto lama dibersihkan ketika memulai sesi event baru untuk membebaskan kuota Safari.
- template dan result dapat membaca metadata event dari sessionStorage atau localStorage.
- Cache service worker dinaikkan agar iPad tidak memakai file event lama.
